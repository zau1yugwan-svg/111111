# 中文转阿语 RTL Figma 插件

这个插件用于把中文界面转换成阿拉伯语 RTL 方向：

- 选中一个或多个 Frame / 画板后运行。
- 默认生成一个 `- AR RTL` 副本，原中文画板不动。
- 如果选中的是组件实例或实例内部元素，插件会复制外层实例并转成普通 Frame 后再处理。
- 整体布局会左右镜像，但每个元素不会被 `scaleX(-1)` 翻转。
- Auto Layout 会同步 RTL 化：横向顺序反转、左右 padding 互换、水平对齐方向互换。
- 矢量图容器只做外部位置镜像，不处理内部路径和内部 Frame，避免图标被拆散或二次镜像。
- Group 只作为外层整体参与父级位置镜像，不在 Group 层改内部子节点坐标，避免 Figma 自动重算 Group 边界导致整体漂移。
- 带旋转的图片填充节点不会被单独改 x 坐标，避免裁剪/旋转图片跑出容器。
- 左对齐文本会改成右对齐；原本居中的文本保持居中。
- 中文日期时间会本地转换，例如 `2025/03/05 17:23:10` 会变成 `17/23/10 05/03/2025`。
- 转换前会先识别画板文本并去重，同样的中文文案只请求一次翻译，再批量替换回所有对应文本节点。

## 安装

1. 打开 Figma 桌面端。
2. 进入 `Plugins` -> `Development` -> `Import plugin from manifest...`。
3. 选择本文件夹里的 `manifest.json`：

```text
/Users/apple/Documents/New project/figma-zh-ar-rtl-plugin/manifest.json
```

## 使用

1. 在 Figma 中选中需要转换的中文画板。
2. 运行 `Plugins` -> `Development` -> `中文转阿语 RTL`。
3. 保持“生成副本”开启。
4. 翻译服务默认选择 `MyMemory 免注册免费翻译`。
5. 可选填写 Email，然后点击 `保存翻译配置`。
6. 点击“转换选中画板”。

## MyMemory 免注册翻译

默认方案不需要 Azure、不需要信用卡、不需要 Key。

- 不填 Email 也可以直接翻译。
- 填写 Email 后，MyMemory 通常会给更高的免费请求额度。
- 公共免费接口可能有日额度、限流或质量波动，适合轻量使用和快速验证。
- 如果插件提示 `MyMemory 翻译接口返回 429`，通常是请求太频繁或免费额度触发。
- 插件里的 `保存翻译配置` 会把服务选择、Email 或自定义接口保存到当前 Figma 客户端本地。
- 插件里的 `重新填写` 会清空已保存配置。

## 自定义接口

如果翻译服务选择 `不翻译，只做 RTL`，插件只做镜像布局和文本方向调整。

自定义接口按 JSON POST 调用，并会同时发送常见字段，方便兼容 LibreTranslate / DeepL 风格服务。当前 manifest 对开发使用放开了网络域名，所以也可以填你自己的接口：

```json
{
  "q": "中文",
  "text": "中文",
  "source": "zh",
  "target": "ar",
  "source_lang": "ZH",
  "target_lang": "AR",
  "format": "text"
}
```

插件会读取返回里的 `translatedText`、`translation`、`text` 或 `result` 字段作为译文。

## 注意

- Figma 没有真正的“一键完整本地机器翻译”能力，翻译要依赖外部接口或你自己的翻译服务。
- 复杂组件、实例、Smart Animate 原型连线等可能需要人工复查。
- 如果图标本身有方向含义，例如返回箭头，插件不会把图标反向；这符合“元素不翻转”的要求，但阿语界面里这类图标通常需要替换成 RTL 版本。
- 日期时间规则：中文格式 `年/月/日 时/分/秒` 会改为 `时/分/秒 日/月/年`，例如 `2025年3月5日 17:23:10` -> `17/23/10 05/03/2025`。
